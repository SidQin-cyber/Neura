# 登录界面测试指南

## 新的登录界面设计

我们已经重新设计了登录界面，现在具有以下特点：

### 🎨 界面特性
- **全屏覆盖**：登录界面完全覆盖整个页面，包括sidebar和header
- **左右布局**：左侧为登录表单，右侧为品牌展示图片
- **现代化设计**：使用Tailwind CSS和现代UI组件
- **响应式布局**：在大屏幕上显示左右布局，小屏幕上只显示登录表单

### 🔒 安全特性
- **强制认证**：未登录用户无法看到sidebar、header或其他界面元素
- **自动重定向**：未登录用户访问任何页面都会被重定向到登录页面
- **会话管理**：登录成功后自动重定向到首页

### 🎯 界面元素

#### 左侧登录表单
- **品牌标识**：Neura logo和品牌名称
- **标题**：登录到您的账户
- **副标题**：AI驱动的智能招聘平台
- **表单字段**：
  - 邮箱地址输入框
  - 密码输入框
  - 记住我选项
  - 忘记密码链接
- **登录按钮**：主要操作按钮
- **社交登录**：Google和GitHub登录选项（UI展示）

#### 右侧品牌展示
- **背景图片**：招聘相关的专业图片
- **渐变遮罩**：增强文字可读性
- **品牌信息**：
  - 标题：智能招聘，精准匹配
  - 描述：使用AI技术为您找到最合适的候选人，让招聘变得简单高效

### 💻 测试步骤

1. **清除所有cookies和localStorage**
   ```javascript
   // 在浏览器控制台执行
   localStorage.clear();
   document.cookie.split(";").forEach(function(c) { 
     document.cookie = c.replace(/^ +/, "").replace(/=.*/, "=;expires=" + new Date().toUTCString() + ";path=/"); 
   });
   ```

2. **访问任何页面**
   - 访问 `http://localhost:3000/`
   - 访问 `http://localhost:3000/search`
   - 访问任何其他页面

3. **验证重定向**
   - 应该自动重定向到 `/login` 页面
   - 不应该看到sidebar、header或其他界面元素

4. **测试登录界面**
   - 界面应该全屏显示
   - 左侧显示登录表单
   - 右侧显示品牌图片（仅在大屏幕上）
   - 表单字段应该正常工作

5. **测试登录功能**
   - 使用测试账户登录：
     - 邮箱：`admin@neura.com`
     - 密码：`123456`
   - 验证表单验证（空字段、错误格式等）
   - 验证成功登录后重定向

6. **验证认证后界面**
   - 登录成功后应该重定向到首页
   - 应该显示sidebar和header
   - 应该显示完整的应用界面

### 🔧 技术实现

#### 路由保护
- 使用Next.js middleware进行路由保护
- 在 `lib/supabase/middleware.ts` 中实现认证逻辑
- 公共路径：`/login`、`/api/auth`

#### 布局逻辑
- 在 `app/layout.tsx` 中根据用户认证状态显示不同布局
- 已认证：显示sidebar + header + 主内容
- 未认证：只显示登录页面内容

#### 登录页面
- 位置：`app/login/page.tsx`
- 使用client组件实现交互功能
- 集成Supabase认证API

### 📱 响应式设计

- **大屏幕 (lg以上)**：显示左右布局
- **中等屏幕 (md-lg)**：显示登录表单，隐藏右侧图片
- **小屏幕 (sm以下)**：全屏显示登录表单

### 🎨 设计规范

- **颜色方案**：以indigo为主色调
- **字体**：使用Inter字体
- **间距**：遵循Tailwind CSS间距规范
- **组件**：使用shadcn/ui组件库
- **图标**：使用Lucide React图标库

### 🐛 常见问题

1. **页面显示不全**
   - 检查HTML和body标签是否有 `h-full` 类
   - 确保没有其他CSS影响全屏布局

2. **重定向循环**
   - 检查middleware配置
   - 确保公共路径正确配置

3. **样式问题**
   - 检查Tailwind CSS配置
   - 确保所有必要的类都可用

### 🔍 调试信息

如果遇到问题，可以检查：
- 浏览器控制台的错误信息
- Network面板的请求状态
- Application面板的cookies和localStorage
- Supabase认证状态
